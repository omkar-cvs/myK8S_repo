Phase 1: Prerequisites & Container Runtime (containerd)
Run on ALL 3 servers (Master and Nodes)

1) Create instances:
Create 3 AWS EC2 instances with Ubuntu 24.04 and t2.medium.

2) Disable swap and enable IP forwarding (Required for K8s):

Bash
# Disable swap permanently
sudo swapoff -a
sudo sed -i '/ swap / s/^/#/' /etc/fstab

# Load necessary kernel modules
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF
sudo modprobe overlay
sudo modprobe br_netfilter

# Enable IP forwarding
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF
sudo sysctl --system
3) Install and configure containerd:

Bash
sudo apt-get update
sudo apt-get install -y containerd apt-transport-https ca-certificates curl gpg

# Generate default config and configure it to use systemd cgroups
sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml
sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml

# Restart and enable the service
sudo systemctl restart containerd
sudo systemctl enable containerd
Phase 2: Install Kubernetes Components
Run on ALL 3 servers (Master and Nodes)

4) Add the new Kubernetes GPG key and repository:

Bash
# Create the keyring directory
sudo mkdir -p /etc/apt/keyrings

# Download the key for Kubernetes v1.33 (you can change this to match your desired version)
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.33/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

# Add the repository
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.33/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list


5) Install kubeadm, kubelet, and kubectl:

Bash
sudo apt-get update
sudo apt-get install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
sudo systemctl enable --now kubelet
Phase 3: Cluster Initialization
Run on MASTER only

6) Set hostname and initialize the cluster:

Bash
sudo hostnamectl set-hostname master
sudo kubeadm init --pod-network-cidr=10.244.0.0/16
(Note: Save the kubeadm join command printed at the end of the init output!)

7) Configure kubectl for the root/ubuntu user:

Bash
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
8) Apply the Pod Network (Flannel):

Bash
# The raw.githubusercontent.com link for flannel has also been updated
kubectl apply -f https://github.com/flannel-io/flannel/releases/latest/download/kube-flannel.yml
Phase 4: Join the Cluster
Run on NODES only

9) Set hostnames for the worker nodes:

Bash
sudo hostnamectl set-hostname node01 # Use node02 on the second server
10) Join the cluster:
Paste the command that was generated back in Step 6. It will look exactly like your old step:

Bash
sudo kubeadm join <MASTER-IP>:6443 --token abcdef.1234567890abcdef \
	--discovery-token-ca-cert-hash sha256:1234..cdef
Phase 5: Verification
Run on MASTER only

11) Check the status:

Bash
kubectl get nodes
kubectl get pods -A
